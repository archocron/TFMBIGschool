El Problema en Industria 4.0

TIEMPO
Inspección Manual Lenta
Operadores humanos revisan pieza a pieza, generando cuellos de botella en la línea de producción.
ERROR
Tasa de Error Variable
La fatiga y la subjetividad visual provocan falsos negativos que escapan al control de calidad.
COSTE
Coste Operativo Alto
Cada operador dedicado a QA representa un coste fijo y una dependencia de personal especializado.
Objetivo: Construir un sistema de inspección visual automático, autónomo y desplegable que demuestre competencias en desarrollo de software, IA, hardware industrial y calidad de código.

Descripción General del Proyecto

Conveyor Camera Monitor es un sistema de inspección visual automática para líneas de producción industriales. Integra un PLC industrial, visión por computador, un modelo de IA entrenado por mí mismo con detección de anomalías (PatchCore) y una interfaz web de control en tiempo real. Además, todo el desarrollo ha sido potenciado con IA generativa (Kimi K2.6) como asistente de código.

Sensor fotoelectrico detecta pieza
↓

↓

↓
Decisión: OK (reanuda) o NG (parada hasta revisión)

Hardware Industrial PARTE I

M-Duino 21+
PLC de Industrial Shields basado en Arduino Mega 2560 con Ethernet W5500 integrado. Servidor Modbus TCP.
Sensor I0.0
Interruptor fotoeléctrico cuadrado, sensor infrarrojo de supresión de fondo, GTB6-DC30. Respuesta en milisegundos.
Relé Q0.0
Relé intermedio delgado tipo interfaz ENMG HF41F-24-ZS, CC 5/12/24 V, montaje en carril DIN. HIGH = PARADA.
Fuente 24V
MZMW fuente conmutada en carril DIN (MDR-10W a 100W). SMPS 10A AC-DC salida única.
Cinta Transportadora
Banda de PVC con motorreductor de 24 V. Transporta placas de ratón bajo la cámara de inspección.
Placas de Ratón
Piezas de prueba reales: placas OK (sin defectos) y NG (arañazos/manchas simulados).

Hardware de Visión PARTE II

Webcam Industrial 4K
4K 8MP IMX415 USB, lente varifocal 5-50 mm. Alta resolución para inspección con OpenCV.
Cámara IP PoE HD
PoE digital 4K con audio HD, lente varifocal 2,8-12 mm / 5-100 mm. Transmite H265/HEVC vía RTSP.
Switch PoE
Cudy GS105P - 5 puertos Gigabit (4 PoE+ 36 W). Plug and play, puertos metálicos blindados.
USB - Ethernet 2.5G
Loydia adaptador USB-C a Ethernet 2.5G / USB 3.0. LAN RJ45 2500 Mbps dedicada al PLC.
Conectividad: Red link-local dedicada PC-PLC (169.254.241.143 → 169.254.241.100) para evitar interferencias y garantizar latencia mínima en Modbus TCP.

Flujo de Producción Paso a Paso

1. El sensor I0.0 detecta flanco ascendente. El Arduino verifica cooldown y estado libre.
2. El PLC PARA la cinta inmediatamente (Q0.0 = HIGH).
3. Arduino publica coil 0 = 1 (pieza detectada) vía Modbus TCP.
4. Backend detecta coil 0 = 1, captura imagen 4MP y ejecuta inferencia con modelo PatchCore entrenado por mí en GPU RTX 4070.
5. OK (IA): El modelo decide OK. Backend escribe coil 1 = 1. Arduino arranca cinta automáticamente.
6. NG (IA): El modelo decide NG. Backend no escribe nada. La cinta permanece PARADA para revisión humana.
7. Operador revisa pieza y pulsa "Continuar" en el frontend.
8. Backend escribe coil 1 = 1. Arduino arranca cinta y espera a que la placa salga (I0.0 = LOW).
9. Cooldown de 2 segundos antes de aceptar la siguiente pieza. Evita doble detección.

Inteligencia Artificial

Detección de Anomalías con Anomalib (Intel) + IA Generativa

¿Qué es Anomalib?
Framework Intel para detección de anomalías visuales. Entrena modelos unsupervised / one-class sin ejemplos de piezas defectuosas.
Modelo: PatchCore
Basado en Wide ResNet-50 v2. Extrae feature maps locales, construye memory bank de patches y detecta desviaciones con precisión milimétrica.
Entrenamiento Real
Modelo entrenado por mí con dataset propio OK/NG. Calibración: media OK 28.1, NG 60.7, umbral 40.92, AUROC sklearn.
Inferencia GPU
Inferencia en RTX 4070 + CUDA 12.4 vía PyTorch. 50-100 ms por imagen, decisiones en tiempo real sobre la cinta.
Heatmaps Automáticos
Mapas de calor automáticos sobre la imagen original para visualizar zonas anómalas al operador.

Infraestructura & Docker

MediaMTX
Servidor de streaming ligero que recibe RTMP y sirve WebRTC. Permite ver la cámara IP en el frontend con latencia inferior a 1 segundo.
FFmpeg
Puente RTSP (cámara IP Hikvision) → RTMP (MediaMTX). Transmisión continua con copia de codecs (-c copy) para mínima carga de CPU.
Docker Compose
Orquestación de contenedores para MediaMTX y Nginx proxy. Aislamiento de servicios de red y fácil replicación del entorno CCTV.
Scripts Windows
start-all.bat levanta backend + MediaMTX + FFmpeg + abre navegador.
stop-all.bat detiene todos los procesos limpiamente.
docker-compose.yml servicios:
- mediamtx: bluenviron/mediamtx (RTMP in / WebRTC out)
- nginx: proxy web estático y redirección de puertos

Buenas Prácticas & Seguridad

Configuración por Entornos
IPs, puertos y rutas externalizadas en config.py. Fácil adaptación a nuevas redes sin tocar lógica de negocio.
Separación de Secretos
Aunque no hay login de usuarios, se evita hardcodear credenciales. La cámara IP usa RTSP sin password expuesto en código fuente.
Thread-Safety
Uso de locks y estados atómicos en el hilo de polling Modbus y en la máquina de estados de producción.
Cooldowns & Debounce
Cooldown de 1.5 s entre triggers y 2 s entre piezas. El PLC ignora el sensor hasta que la placa anterior salga físicamente.
Resiliencia
Si el PLC no responde, el backend sigue operativo en modo manual. Si la webcam falla, se reporta sin crashear el servidor.
Documentación Dual
README.md para humanos (instalación, uso) y AGENTS.md para desarrolladores/agentes IA (contexto técnico detallado).
